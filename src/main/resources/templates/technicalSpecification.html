Technical specification

Architecture Overview

DevTeam is a classical multi-layered Spring Boot web application following:
  • MVC architecture
  • Role-based access control (RBAC)
  • Server-side rendering via Thymeleaf
  • Relational storage with Flyway migrations
  • MySQL running in Docker via docker-compose (Spring Boot Docker Compose integration)
The system is built with clean separation of concerns:
  • Presentation layer: HTML templates + controllers
  • Application layer: services, business logic
  • Persistence layer: JPA entities + repositories
  • Database layer: MySQL schema managed through versioned migrations

Infrastructure & Build

Build Tools
  • Maven 3.x
    • spring-boot-maven-plugin (including support for build-image via buildpacks)
    • maven-surefire-plugin for tests
    • maven-failsafe-plugin for integration tests
CI/CD
  • GitHub Actions as primary CI environment
  • On each push / pull request:
    • mvn verify with all automated tests
    • Publishing JUnit test reports
    • Static code analysis & quality gate via Sonar
  • Build artifacts:
    • Executable Spring Boot JAR for local / demo runs
    • (Optional) Docker image build step for containerized deployment
Dockerization
  • App runs as Spring Boot JAR (no dedicated runtime image yet)
  • MySQL provided by Docker Compose for local development
Hot Reload (Dev Mode)
  • Spring DevTools for auto-restart on classpath changes
Testing
  • JUnit 5
  • Mockito
  • Testcontainers (MySQL) for JPA slice and repository testing
  • Multi-phase test pyramid:
    • Unit tests
    • Web slice tests
    • JPA tests
    • Smoke tests

Deployment Environment

Local Development
  • Docker Compose starts MySQL database
  • Spring Boot runs on the host JVM (IDE / CLI)
  • Flyway migrations execute automatically on application startup
  • Spring DevTools provides hot reload in local development
Prod-like Deployment
  • Application packaged as executable Spring Boot JAR
  • Optional container image via Spring Boot buildpacks (spring-boot:build-image)
  • No external web server required (embedded Tomcat)
  • Configuration via application.properties and environment variables
