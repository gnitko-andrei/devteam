Project overview

DevTeam is a modular project-management system built to support the full lifecycle of internal development projects. It centralizes project data, task tracking, role-based access, developer workload management, and cost monitoring.

Workflow is intentionally streamlined: customers define project parameters, managers coordinate execution, and developers report progress and time spent. All operations follow strict role-based authorization to ensure clear responsibility boundaries.

The platform works as a lightweight alternative to enterprise PM tools, with emphasis on clarity, predictable behavior, and low friction for end users. Data flow and UI are optimized for small-to-medium teams working in predictable delivery cycles.

Project Workflow

Project creation

Customer initializes a new project by defining core metadata (name, specification) and assigning a manager.

Task lifecycle
  • Customer/Manager define initial task set.
  • Developer moves tasks through lifecycle: NEW → IN_PROGRESS → DONE.
  • Developer logs time during IN_PROGRESS phase.
Budget accumulation

Project cost is calculated automatically based on logged work time × developer hourly rates.

Project oversight
  • Customers monitor status and specification.
  • Managers adjust status, developers, and validate task completion.
  • Admins supervise all users and resolve account-level issues.

User Roles

  • Sign in / Sign up into the system.
  • View personal profile and edit personal data.
  • Update the password.
  • Delete the account.

  • Access the admin panel with a list of all user accounts.
  • Assign and revoke roles (USER, CUSTOMER, MANAGER, DEVELOPER).
  • Perform account cleanup and system maintenance tasks.

  • Create new projects (name, specification, manager assignment).
  • Edit project core metadata.
  • View project budget spent and developer list.
  • Create new tasks for projects they own.
  • Review all tasks within their projects.
  • Delete entire project when necessary.

  • Update project status (NEW / IN_PROGRESS / DONE / CANCELLED).
  • Assign or remove developers from a project.
  • Delete entire project when necessary.
  • Review all tasks within managed projects.
  • Create tasks or edit existing ones.
  • Validate or reject task completion.
  • Monitor workload distribution and cost accumulation.
  • Moderate project changes created by customers.
  • Manage developers' hourly rates.

  • View relevant project information (manager, customer, specification).
  • View tasks assigned within active projects or create new tasks.
  • Change task status: NEW → IN_PROGRESS → DONE (when permitted).
  • Log work hours directly into the task.
